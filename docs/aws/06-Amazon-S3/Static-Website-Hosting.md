# 04. Hosting Static Website in S3

Amazon S3 can be used not only for storing data but also for **hosting static websites**.

A **static website** typically includes HTML, CSS, JavaScript, and image files ‚Äî no backend logic or server processing.

When configured properly, S3 serves these files over the internet with a **public website endpoint**.

---

### üß© Key Concepts

| Concept                    | Description                                                                 |
| -------------------------- | --------------------------------------------------------------------------- |
| **Static Website Hosting** | Enables your S3 bucket to act as a web server for hosting static content.   |
| **Website Endpoint**       | A public URL automatically generated by S3 once static hosting is enabled.  |
| **Index Document**         | The homepage (e.g., `index.html`) displayed when users visit the site root. |
| **Bucket Policy**          | Defines public access permissions for your bucket‚Äôs content.                |
| **Public Read Access**     | Required to make your website visible to everyone on the internet.          |

---

## ‚öôÔ∏è Steps to Host a Static Website on S3

### **Step 1 ‚Äî Create an S3 Bucket**

1. Go to the **AWS Management Console ‚Üí S3**.
2. Click **Create bucket**.
3. Enter a **unique bucket name** (e.g., `my-coffee-website`).
4. Choose a **region** (e.g., `ap-south-1`).
5. Under **Block Public Access settings**, **uncheck** ‚ÄúBlock all public access‚Äù.
6. Confirm acknowledgment ‚Üí Click **Create bucket**.

### **Step 2 ‚Äî Upload Website Files**

1. Open your bucket.
2. Click **Upload** ‚Üí **Add files**.
3. Upload:
   - `index.html`
   - `coffee.jpg`
   - `beach.jpg`
4. Click **Upload**.

### **Step 3 ‚Äî Enable Static Website Hosting**

1. Go to the **Properties** tab of your bucket.
2. Scroll to **Static website hosting** ‚Üí Click **Edit**.
3. Choose **Enable**.
4. Select **Host a static website**.
5. In **Index document**, enter: **index.html**
6. Click **Save changes**.

### **Step 4 ‚Äî Make Files Public**

1. Go to the **Permissions** tab.
2. Under **Bucket policy**, click **Edit**.
3. Add the following JSON policy:
   ```json
   {
     "Version": "2012-10-17",
     "Statement": [
       {
         "Sid": "Statement1",
         "Effect": "Allow",
         "Principal": "*",
         "Action": "s3:GetObject",
         "Resource": "arn:aws:s3:::himanshupapola-demo-s3-v1/*"
       }
     ]
   }
   ```
4. Save the policy.

### **Step 5 ‚Äî Access Your Website**

1. Return to the **Properties** tab.
2. Scroll down to **Static website hosting**.
3. Copy the **Bucket website endpoint** (URL).
4. Paste it into your browser.

   Example URLs:

   - `http://himanshupapola-demo-s3-v1.s3-website.ap-south-2.amazonaws.com`

## üß† Troubleshooting

| Issue                    | Cause                            | Fix                                                           |
| ------------------------ | -------------------------------- | ------------------------------------------------------------- |
| **403 Forbidden Error**  | Bucket or objects are not public | Ensure bucket policy allows `s3:GetObject` for `Principal: *` |
| **Index not loading**    | Missing `index.html` file        | Upload or rename your homepage file to `index.html`           |
| **Image not displaying** | Wrong file path or permissions   | Verify correct object key and public access                   |

---
## ‚úÖ Summary

- Amazon S3 can **host static websites** (HTML, CSS, JS, images).
- Enable **Static website hosting** under **Properties**.
- Ensure **public read access** via a bucket policy.
- Access your site using the **S3 website endpoint URL**.
